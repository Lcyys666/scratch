特殊云变量行为

info

本页面是关于 TurboWarp 打包器 的说明。

默认禁用的"特殊云变量行为"选项会改变特定命名的云变量的行为，以为您的项目解锁新的兼容性。这是基于 HTMLifier 中的类似功能。此功能可以在"云变量"部分启用。

要创建这些变量，只需像平常一样创建云变量，但要使用下面列出的特定名称。例如，要使用 ☁ url 变量，创建一个名为 url 的变量并将其标记为云变量。

启用特殊云变量行为将覆盖这些变量的任何其他设置，因此像 ☁ username 这样的变量永远不会本地存储或与其他用户同步。

☁ url​

☁ url 的值将被设置为页面的当前 URL。更改 ☁ url 的值不会产生任何效果。

☁ redirect​

当 ☁ redirect 的值被设置为一个 URL 时，当前标签页将导航到该 URL。

☁ open link​

当 ☁ open link 的值被设置为一个 URL 时，项目将尝试打开一个带有该 URL 的新标签页。请注意，由于大多数浏览器内置的弹出窗口拦截器，这并不总是可靠的。

☁ username​

当 ☁ username 的值改变时，侦测类别中的 username 积木的值也会改变。

☁ pasted​

当用户使用快捷键（如 ctrl+v）在页面上粘贴一些文本时，该文本会被存储在 ☁ pasted 中。

☁ set clipboard​

当 ☁ set clipboard 的值改变时，页面将尝试将文本存储到用户的剪贴板中。这可能并不总是有效。

☁ room id​

当 ☁ room id 的值改变时，用于同步云变量的项目 ID 也会改变。例如，如果项目的原始 ID 是 1234，而 ☁ room id 被设置为 xyz，新的项目 ID 将是 1234-xyz。要将项目 ID 重置为原始 ID，请将 ☁ room id 的值设置为空字符串。

这可以作为一种为云变量项目添加服务器选择器的方法，而无需创建大量额外的变量。只有具有相同房间 ID 的人之间才会同步变量。云变量可能需要几秒钟才能重新开始工作，因为它需要重新连接到云变量服务器。

房间 ID 不会影响本地存储的云变量。

☁ eval​

warning

此选项需要启用"不安全的特殊云变量行为"。

不安全的云变量行为允许打包的项目在项目通常执行的"沙盒"之外执行任意代码。根据您打包的环境，这会授予项目对您计算机的完全控制权，包括安装病毒的能力。

如果您不信任正在打包的项目或不使用此功能，请关闭此选项。

当 ☁ eval 的值改变时，其值将作为 JavaScript 代码执行。

如果 JavaScript 代码成功执行，其输出将被存储在 ☁ eval output 中。

如果执行 JavaScript 时出现错误，错误将被存储在 ☁ eval error 中。

如果 JavaScript 返回一个 Promise，如果 promise 成功解决，它将在 ☁ eval output 中存储解决的值；如果 promise 被拒绝，则在 ☁ eval error 中存储错误。请注意，设置 ☁ eval 始终是一个即时过程，因此输出变量可能不会立即更新。
