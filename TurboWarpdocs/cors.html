CORS：为什么您的 fetch 积木块不起作用

任何使用过 Fetch、HTTP、Network 和类似扩展的人都注意到，当您尝试获取某些网站时，即使按下"允许"后，积木块仍然不起作用：

这个问题的原因是 CORS，这是一个限制网站之间交互方式的安全特性。

什么是 CORS？​

它代表跨源资源共享（Cross-Origin Resource Sharing），但我们不会深入讨论 内部细节 -- 其他地方已经详细说明了。CORS 解决的核心问题很容易理解：

  • 一些网站想要阻止其他网站访问
  • 其他网站想要允许其他网站访问

想象一下，如果您访问的任何网站都能像您一样访问您的银行网站。那将是非常糟糕的！但有时这种访问并不是问题，实际上是预期的。Scratch API 的部分功能启用了 CORS，这就是 TurboWarp 如何从 Scratch 加载项目的方式。

CORS 是网站声明是否允许其他网站访问它们的方式。默认情况下，访问是不允许的。网站必须选择加入 CORS 才能让其他网站访问它们。如果网站没有选择加入，您的浏览器会给 TurboWarp 一个非常通用的"网络错误"。

如何修复您的积木块​

这取决于 URL 的用途。

  • 切换到不同的 URL： 如果 URL 只是用于托管静态文件，请找到另一个支持直接下载和 CORS 的主机。如果一个 API 不支持 CORS，请检查竞争对手的 API 是否支持。
  • 使用 CORS 代理： 与其让您的浏览器直接访问网站，不如让另一个服务器（称为 CORS 代理）代表您访问该网站，然后返回响应但允许 CORS。网上有很多公共 CORS 代理，但它们往往寿命很短，因为运营成本高且经常被滥用。CORS 代理还可以监视您项目的网络请求，并篡改它们。这可能会带来问题。TurboWarp 目前不运行自己的 CORS 代理。
  • 切换到 TurboWarp Desktop： 桌面应用程序有一个绕过 CORS 的选项。见下文。

桌面应用程序​

在 TurboWarp Desktop 中，有一个选项可以绕过 CORS 以允许访问任何网站。出于安全考虑，默认情况下它是禁用的，以匹配普通网页浏览器。在左上角的设置按钮下打开桌面设置（在旧版本中，桌面设置在右上角的 ? 按钮下）：

当桌面设置窗口打开时，勾选"允许扩展访问任何网站"的复选框。

打包的项目​

在浏览器中运行的打包项目受到与其他任何网站相同的限制。

打包为 Electron 应用程序的项目默认绕过 CORS，类似于桌面应用程序中的选项。

如果您正在运行服务器​

如果您希望网站能够访问您的网站，请在您想要公开的每个响应中将 Access-Control-Allow-Origin 头部设置为 *。如果您搜索您使用的 Web 服务器或框架的名称加上"cors"，您应该能找到很多示例。
