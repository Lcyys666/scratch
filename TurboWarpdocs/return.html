自定义报告器

TurboWarp 现在支持自定义报告器，让您可以使用项目中的任何自定义积木块作为报告器，就像 Scratch 内置的加法积木块一样。这是文本编程语言中的一个相当标准的功能，但一开始可能很难理解。

info

使用此功能的项目不能上传到 Scratch 网站。

从技术上讲，自定义报告器并不能让您做任何以前做不到的事情 -- 它们只是让代码更容易阅读和编写。例如，如果您的项目需要重复文本很多次，您可能会编写这样的脚本：

然后，像往常一样创建自定义积木块。要创建自定义报告器，只需将"return"积木块拖入脚本中 -- 它位于自定义积木块列表的底部。您不需要勾选任何复选框或切换任何选项。

一旦遇到 return 积木块，它的功能类似于"停止这个脚本"。

TurboWarp 会自动将积木块重塑为它猜测的正确形状，但有时它会猜错。您可以通过右键点击积木块，然后选择"更改为堆叠积木块"或"更改为报告器"来随时更改积木块的形状。

同样，如果自定义积木块中的所有 return 积木块都包含布尔值（真/假），如"1 > 2"，那么自定义积木块也会是布尔形状的。这只是一个视觉辅助，因为任何自定义报告器都可以放入任何输入中。

info

使用编译器时，要注意因递归太多而导致的栈溢出错误。递归超过几千次通常会导致错误。

编译器和解释器之间的区别​

解释器支持无限递归，而编译器受 JavaScript 栈大小的限制，通常约为几千次调用，但在不同系统和浏览器之间会有所不同。

info

建议确保自定义报告器只运行某些算法并输出最终答案。为了确保您的项目在编译器和解释器中运行相同，自定义报告器应该避免移动精灵、说文本、等待等操作。

为了提高性能，编译器有一个称为短路求值的功能。想象一个 Scratch 积木块如"< (1 = 2) and (3 = 3) >"。当编译器看到这个积木块时，它首先计算"1 = 2"，这显然是假的。在这种情况下，编译器甚至不需要检查"and"积木块的另一边，因为这无关紧要：最终结果将始终为假。通常这只是免费的性能提升，但如果您的积木块有副作用，如移动精灵，由于解释器总是计算所有积木块，会出现不同的行为。试试这个脚本来看看：
