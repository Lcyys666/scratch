修改入门

这些是为 TurboWarp 本身设置开发环境的说明。如果您想向我们提交拉取请求或制作自己的修改版本，这会很有用。

如果您只想开发自定义扩展，请参阅自定义扩展文档。

依赖项​

我们所有的包都需要 Git 和 Node.js v22（v18 或更高版本可能都可以工作，但我们不能保证）。我们假设您熟悉命令行。

某些包可能需要安装一些额外的东西，所以请查看您想要开发的每个包中的 README。

TurboWarp 是一个大型应用程序，可能需要多个 GB 的磁盘空间和内存来构建。

关于 Scratch 组织结构的说明​

Scratch 被分成许多不同的包，每个包实现应用程序的一部分。

  • scratch-gui 实现了大部分界面（例如精灵列表），连接所有内容，并且是插件所在的位置
  • scratch-vm 运行项目。编译器就在这里。
  • scratch-render 用于显示舞台、精灵、对话气泡和画笔等内容。它还实现了像"碰到"这样的积木块。请注意，在精灵上方渲染的内容（如变量监视器）实际上是 scratch-gui 的一部分。
  • scratch-svg-renderer 帮助修复各种 SVG 渲染问题（我们将其重命名为 @turbowarp/scratch-svg-renderer）
  • scratch-render-fonts 包含 SVG 造型可以使用的所有字体
  • scratch-paint 是造型编辑器
  • scratch-parser 提取和验证 sb2 和 sb3 文件
  • scratch-storage 是一个围绕 fetch() 的抽象，用于下载（理论上也可以上传）文件（我们将其重命名为 @turbowarp/scratch-storage）
  • scratch-l10n 包含一些翻译（我们将其重命名为 @turbowarp/scratch-l10n）

此外，桌面应用程序和打包器也是支持仓库。

构建 GUI​

如果您想修改 Scratch，您需要能够构建 GUI。这是开发 Scratch 包时常用的模式：

# 克隆仓库
git clone https://github.com/TurboWarp/scratch-gui
cd scratch-gui

# 安装依赖项（比 `npm install` 更好，因为它更快且不会修改 package-lock.json）
npm ci

# 启动开发环境
npm start

这会为大多数包启动实时开发服务器（如果有的话）。例如，对于 scratch-gui，可以在 http://localhost:8601/ 访问开发环境。有关其他包的信息，请参阅 README 或 npm start 的输出。

构建​

虽然 npm start 对开发很有用，但在某些时候您需要获取原始文件。您可以使用以下命令：

npm run build

输出将在 build 文件夹中。

在将 TurboWarp 部署到实际网站时，您应该启用生产模式。这将导致更快的执行速度和大大减少的文件大小：

# mac, linux
NODE_ENV=production npm run build

# windows 命令提示符（未测试）
set NODE_ENV=production
npm run build

# windows powershell
$env:NODE_ENV="production"
npm run build

默认情况下，TurboWarp 生成类似 https://turbowarp.org/editor.html#123 的链接。但是，通过设置变量 ROOT=/ 和 ROUTING_STYLE=wildcard（与设置 NODE_ENV=production 的方式相同），您可以获得类似 https://turbowarp.org/123/editor 的路由。请注意，这需要一个能够设置适当别名的服务器。scratch-gui 中的 webpack 开发服务器已经设置好了这个。对于生产环境，您可能需要类似 https://github.com/TurboWarp/turbowarp.org 的东西。

链接其他包​

要开发 scratch-gui 以外的包，您需要告诉 npm 使用包的本地副本，而不是从互联网下载的副本。这称为链接。模式是：

# 在与 scratch-gui 相同的文件夹中克隆您想要开发的包
# （文件夹位置其实无关紧要，但这样更容易跟踪）
cd scratch-gui/..
git clone https://github.com/TurboWarp/scratch-vm

# 在子包中安装依赖项
cd scratch-vm
npm ci

# 告诉 npm 这是您的包的本地副本
npm link

# 对于某些包（例如 storage、svg-renderer），您可能需要构建才能应用更改
npm run build

# 告诉 scratch-gui 使用您的包的本地副本
cd ../scratch-gui
npm link scratch-vm
