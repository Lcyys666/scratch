更好的开发服务器

Python 内置的 HTTP 服务器使用简单，但很原始。我们可以做得更好。这部分是完全可选的，但我们认为它会让您的工作更轻松。

TurboWarp/extensions​

extensions.turbowarp.org 的 GitHub 仓库是 TurboWarp/extensions。这里提供的其中一个东西是我们使用的开发服务器，以及构建网站的代码。您也可以在本地运行它。

这需要在本地安装 Git 和 Node.js。运行以下命令来下载服务器并安装其依赖：

git clone https://github.com/TurboWarp/extensions.git
cd extensions
npm ci

现在打开您的代码编辑器，进入您刚刚下载的 "extensions" 仓库。

关闭您的旧开发服务器，并使用以下命令启动新的服务器：

npm run dev

现在导航到 http://localhost:8000 -- 您应该看到一个类似于 extensions.turbowarp.org 的列表。现在，我们不会关心列表，我们只是要使用开发服务器部分。

保存扩展 JS 的位置是仓库内的 extensions 文件夹。如果您打算稍后将这些提交给我们，您应该为您的扩展创建一个用户文件夹。例如，如果您的用户名是 TestMuffin，您可能会将您的获取扩展保存在 extensions/TestMuffin/fetch.js 文件夹中，并使用 http://localhost:8000/TestMuffin/fetch.js 加载它。然后扩展也应该使用包含您用户名的 ID，比如 testmuffinfetch。

本地开发服务器设置了正确的头部，这样您就不需要手动硬刷新来确保您的脚本更改生效，并且它尽可能地模仿真实的 extensions.turbowarp.org 网站。它还包括许多常见 Scratch API 的 TypeScript 提示和用于常见错误的 ESLint 规则。

将来，我们可能会考虑添加更多开发功能。

练习​

  1. 您能弄清楚如何向开发服务器添加新的 HTML 文件吗？（提示：查看 "website" 文件夹）
  2. 您能弄清楚如何将您的扩展添加到主页列表中吗？（提示：查看 "extensions/extensions.json"）
  3. 您能弄清楚如何为您的扩展在主页上添加图片吗？（提示：在 "images" 中创建一个与 "extensions" 中的文件具有相同文件夹和基本名称的图片）
