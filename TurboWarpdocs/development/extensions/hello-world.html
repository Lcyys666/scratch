Hello, world!

让我们从创建一个非常简单的扩展开始。这个扩展只添加一个返回 "Hello, world!" 的积木块：

hello-world.js - Try this extension
class HelloWorld {
  getInfo() {
    return {
      id: 'helloworld',
      name: 'It works!',
      blocks: [
        {
          opcode: 'hello',
          blockType: Scratch.BlockType.REPORTER,
          text: 'Hello!'
        }
      ]
    };
  }

  hello() {
    return 'World!';
  }
}

Scratch.extensions.register(new HelloWorld());

上面是我们将用于展示扩展代码的标准组件。请注意标题旁边的"试用此扩展"按钮 -- 该链接可让您在不需要在本地操作的情况下查看此扩展的功能。请注意，这些扩展主要用于演示 API 功能；它们并不是真正用于项目中的。在 extensions.turbowarp.org 上几乎总是有另一个扩展可以做同样的事情，而且做得更好。

如果您只是使用简单的文件来开发扩展，请将此代码保存到名为 "hello-world.js" 的文件中。如果您使用的是本地 HTTP 服务器，请将代码保存在服务器可以访问的名为 "hello-world.js" 的文件中。

现在，转到 TurboWarp 编辑器，点击添加扩展按钮（2个积木块旁边的 +），滚动到 Scratch 部分的底部，选择"自定义扩展"选项。输入您的本地 HTTP 服务器的完整 URL，或使用其他选项卡选择您的文件或粘贴代码。现在，不要勾选"在沙盒外运行扩展"框。

一秒钟后，侧边栏中应该出现一个名为"It works!"的扩展。如果没有出现，请打开开发者工具并查看控制台中是否有任何警告。一些最常见的错误是：

  • JavaScript 语法错误。这应该会在浏览器的开发者工具中显示。
  • JavaScript 运行时错误。这应该会在浏览器的开发者工具中显示。
  • 您的广告拦截器或浏览器正在阻止对 localhost 的请求。尝试关闭广告拦截器。一旦您的扩展发布在面向互联网的网站上，这就不应该是问题了。

现在，我们将按照运行顺序分析这个文件中发生的事情。

构造和注册​

class MyExtension {

这是一个标准的 JavaScript 类。按照惯例，我们以类的形式定义扩展。类的名称并不重要，但我们建议基于扩展的名称来命名。在这个阶段，它不需要是唯一的。

Scratch.extensions.register(new HelloWorld());

这将您的类构造成一个对象，并引入允许扩展功能的特殊 API：Scratch。Scratch 上有很多内容，但最重要的函数之一是 Scratch.extensions.register。

确保始终只调用一次 register()。如果不调用它，您的扩展将永远不会被添加，我们将一直等待它加载。如果多次调用它，行为是未定义的，所以不要依赖它。

getInfo()​

  getInfo() {
    return {
      id: 'helloworld',
      name: 'It works!',
      blocks: [
        {
          opcode: 'hello',
          blockType: Scratch.BlockType.REPORTER,
          text: 'Hello, world!'
        }
      ]
    };
  }

当您调用 register() 时，Scratch 将调用提供对象的 getInfo() 函数。此对象必须返回一个包含扩展元数据的对象。一些字段是 id、name 和 blocks。以下是这些字段的简要总结：

类型描述
idstring仅由此扩展使用的唯一 ID。多个扩展不能共享相同的 ID。只能使用字母 a-z 和数字 0-9 -- 不能使用空格或特殊字符。
namestring在工具箱中显示的扩展名称。如果未提供，将默认为扩展 id。
blocksarray描述项目中积木块的对象列表。

以下是每个积木块应该具有的内容的简要总结：

类型描述
opcodestring当此积木块运行时应运行的函数的名称。例如，如果是 "hello"，则将运行类的 "hello" 方法。每个 opcode 在每个扩展中必须是唯一的，所以多个扩展可以各自有一个 opcode 为 "hello" 的积木块。
blockTypeScratch.BlockType.*决定积木块的形状。最常见的是 Scratch.BlockType.COMMAND、Scratch.BlockType.REPORTER 或 Scratch.BlockType.BOOLEAN。详见下表。
textstring积木块在编辑器中显示的文本。这里有一些特殊语法用于处理参数，将在下一部分讨论。
argumentsobject可选。在下一部分讨论。

以下是 blockType 的可接受值：

描述示例
Scratch.BlockType.COMMAND一个不返回值的积木块移动 10 步
Scratch.BlockType.REPORTER一个返回字符串或数字的圆形积木块x 坐标、造型名称
Scratch.BlockType.BOOLEAN一个带有尖角的返回布尔值（真或假）的积木块鼠标按下
Scratch.BlockType.HAT一个响应特定条件开始的积木块。稍后讨论。当音量 > 10
Scratch.BlockType.EVENT一个专门响应事件开始的积木块。稍后讨论。当角色被点击

虽然还存在其他 BlockType，但它们不能很好地工作，此时不会讨论。

积木块​

  hello() {
    return 'Hello, world!';
  }

这定义了将运行 opcode 为 "hello" 的积木块的函数。在这种情况下，我们的积木块非常简单，只返回一个字符串。REPORTER 积木块应该返回字符串、数字或布尔值，而 BOOLEAN 积木块应该只返回布尔值。请注意，null、undefined、列表和对象不是这些积木块的预期返回值。

当您想要更改扩展时，只需修改扩展并重新加载页面。这里有一个让您的工作更轻松的提示：您可以使用 ?extension= URL 参数自动加载扩展，而不需要进入库。例如，如果您的扩展 URL 是 http://localhost:8080/hello-world.js，您可以使用 https://turbowarp.org/editor?extension=http://localhost:8080/hello-world.js 自动加载它。

如果您发现刷新时更改没有被应用，请尝试使用浏览器的"硬刷新"或"不使用缓存刷新"快捷键。

练习​

  1. 将 "Hello!" 积木块改为返回您最喜欢的数字。记得相应地重命名积木块。
  2. 更改积木块的 opcode。
  3. 添加另一个积木块。使这个积木块使用 type 为 Scratch.BlockType.BOOLEAN 并随机返回一个布尔值（真或假）。
