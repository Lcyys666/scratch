自定义扩展简介

自定义扩展是一种使用 JavaScript 为 TurboWarp 添加新积木块的方法。

在本教程中，我们将介绍如何构建自定义扩展。您可以使用侧边栏或每个页面底部的链接来浏览这些页面。我们假设您在进入下一页之前，已完整阅读了每一页并完成了列出的练习（如果有的话）。

info

它使用了完全不同的结构和风格。我们希望它更容易理解。这个教程可能存在一些错误和遗漏。让我知道您的想法。谢谢！

当人们提到"扩展"时，他们可能指的是以下几种：

可以访问 Scratch 内部功能可以通过 URL 加载
核心扩展（画笔、翻译等）✅❌
沙盒化自定义扩展❌✅
非沙盒化自定义扩展✅✅

这些部分的文档仅涉及自定义扩展。虽然核心扩展共享许多基本原理，但开发它们的过程有很大不同。有关构建核心扩展的起点，请参阅入门指南。

我们将在稍后讨论沙盒化和非沙盒化扩展之间的区别。

兼容性​

自定义扩展与 Scratch 不兼容。使用自定义扩展制作的项目不能上传到 Scratch 网站。但是，它们可以使用 TurboWarp 打包器 进行打包。

前提条件​

自定义扩展开发需要 JavaScript 编程知识。如果您不熟悉 JavaScript，请先学习它。您可以使用您喜欢的搜索引擎找到学习资源。如果您不知道像 "1" 和 1 之间的区别这样的基础知识，开发扩展将会非常困难。作为志愿者，我们没有太多时间帮助您学习 JavaScript -- 抱歉。

扩展可以使用网站或桌面应用程序进行开发。

我们假设您可以访问浏览器或桌面应用程序内置的开发者工具。通常可以通过右键点击 > 检查元素来访问。在桌面应用程序中，可以使用 Ctrl+Shift+I（在 macOS 上是 Option+Command+I）显示。在没有开发者工具的情况下编写 JavaScript 是极其痛苦的，我们无法为此提供帮助。

教程结构​

本教程采用自下而上的方法。我们将从最基本的、实际上毫无用处的扩展开始，逐步构建到更有用的内容。

我们知道有些人会急于开始分享他们的扩展，但是我们要求您在发布扩展或提交给我们之前阅读完整个教程，这样您分享的扩展才会真正有用。

准备开发环境​

在最新版本的 TurboWarp 中，有几种开发扩展的方法。

文件（最简单）​

最新版本的 TurboWarp 网站和桌面应用程序在自定义扩展菜单中有一个选项，可以从本地文件或从复制和粘贴的 JavaScript 代码加载扩展。这将是最简单的开发扩展的方式，因为它只需要一个文本编辑器就可以在任何计算机上完成。

本地 HTTP 服务器（推荐）​

但是，如果可能的话，您应该使用本地 HTTP 服务器，让 TurboWarp 从您的计算机获取扩展。这样可以加快开发过程，因为每次更改时您都不必在 TurboWarp 中选择/粘贴 JavaScript 代码。有很多选项可以安装这样的服务器。如果您已经安装了 Python，您就已经有一个了：

cd path/to/where/you/will/store/your/extensions
python3 -m http.server 8080

这将在您运行该命令的文件夹中启动一个本地 HTTP 服务器，地址为 http://localhost:8080/。

我们将最终介绍官方开发服务器，但我们建议现在从最基本的设置开始。

目前，您应该使用除 8000 以外的端口。我们稍后会详细讨论这一点，但目前我们希望您编写的扩展在沙盒中运行。在沙盒外运行的扩展有一些额外的责任，我们将在稍后讨论。

要测试您的服务器是否工作，创建一个名为 hello-world.js 的文件并在其中放入任何文本。确保您可以通过访问类似 http://localhost:8080/hello-world.js 的链接在浏览器中读取文件内容。
