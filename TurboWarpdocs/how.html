TurboWarp 如何让 Scratch 项目运行速度提升 10-100 倍

TurboWarp 使用编译器，而 Scratch 使用解释器。这使 TurboWarp 的运行速度比 Scratch 快 10-100 倍（取决于项目），但它使得实时脚本编辑变得不切实际。

测试ScratchTurboWarp
快速排序 200000 项10.746s0.0528s
循环光线追踪 r=1 s=10 dof=.08832s16s

（在 Linux 上的 Chromium 103 中测试）

考虑以下脚本：

Scratch 的解释器在运行时遍历抽象语法树。在内部，它看起来像这样：

每当 Scratch 执行任何积木块时，它都必须做很多事情：

  • 它必须使用积木块的 ID 查找积木块，并确定积木块的操作码对应哪个函数。
  • 如果积木块有输入，那些也是积木块，必须经过与其他任何积木块相同的步骤，更深层的输入也必须如此。
  • 它手动维护积木块、循环、条件、程序等的栈。
  • Scratch 脚本可以被暂停，所以所有这些都必须以可以暂停和稍后恢复的方式发生。
  • Scratch 脚本可以在运行时更改，所以提前缓存所有内容很困难。
  • 等等。在 Scratch 中，即使执行单个积木块，也有很多事情要做。

解释器的开销是建立在 JavaScript 本身的开销之上的。由于这段代码涉及许多动态类型，JavaScript JIT 可能很难优化它。

TurboWarp 的编译器通过将脚本直接转换为 JavaScript 函数来消除所有这些开销。例如，上面的脚本变成：

const myVariable = stage.variables["`jEk@4|i[#Fk?(8x)AV.-my variable"];
return function* () {
  while (true) {
    runtime.ext_scratch3_motion._moveSteps((+myVariable.value || 0), target);
    yield;
  }
};

需要注意的事项：

  • 不再需要查找积木块 ID 或操作码：它就是 JavaScript。
  • 不再需要手动查找输入：它们就是 JavaScript 参数。
  • 不再需要手动维护状态：它就是 JavaScript。
  • 由于这是一个单一的 JavaScript 函数，我们不能轻松实现实时脚本编辑
  • 如果 JavaScript JIT 注意到某个变量总是数字，理论上它可以相应地优化。
  • 与典型的人工编写的 JavaScript 相比，这个 JavaScript 看起来很奇怪，运行速度也较慢，因为我们保持了与边缘情况 Scratch 行为的兼容性。
  • 我们手动格式化了 JavaScript 并重命名了一些变量以使其更易读。实际代码使用像 b0 这样的变量名，并且没有格式化。

当然，这是一个非常简单的脚本，其中解释器的开销可以忽略不计，这是大多数项目的情况。只有当您每帧执行数千个积木块时，解释器的开销才会变得显著。

这是一个更复杂的例子：一个简单的排序算法（冒泡排序）。

const length = stage.variables["O;aH~(njYNn}Bl@}!%pS-length-"];
const list = stage.variables["O;aH~(njYNn}Bl@}!%pS-list-list"];
const newLength = stage.variables["O;aH~(njYNn}Bl@}!%pS-new-"];
const i = stage.variables["O;aH~(njYNn}Bl@}!%pS-i-"];
const temp = stage.variables["O;aH~(njYNn}Bl@}!%pS-tmp-"];
return function fun1_sort () {
  length.value = list.value.length;
  // 重复直到 length = 0
  while (!compareEqual(length.value, 0)) {
    newLength.value = 0;
    i.value = 1;
    // 重复 length - 1 次
    for (var counter = ((+length.value || 0) - 1) || 0; counter >= 0.5; counter--) {
      // 将 i 增加 1
      i.value = ((+i.value || 0) + 1);
      // 如果列表的第 i - 1 项大于列表的第 i 项
      if (
        compareGreaterThan(
          list.value[((((i.value || 0) - 1) || 0) | 0) - 1] ?? "",
          list.value[((i.value || 0) | 0) - 1] ?? ""
        )
      ) {
        // 交换列表的第 i 项和第 i - 1 项
        temp.value = listGet(list.value, i.value);
        listReplace(
          list,
          i.value,
          list.value[((((+i.value || 0) - 1) || 0) | 0) - 1] ?? ""
        );
        listReplace(
          list,
          (+i.value || 0) - 1,
          temp.value
        );
        newLength.value = i.value;
      }
    }
    length.value = newLength.value;
  }
};

实时脚本编辑​

如果您使用编译器启动脚本，您将无法移动、删除或添加积木块，并使更改像在 Scratch 中那样实时反映。脚本必须重新启动才能应用更改。我们认为有一些方法可以让这个功能工作，但它们会损害性能或增加显著的复杂性。这是我们最终想要实现的功能，但不是现在。
